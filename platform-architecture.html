Request-to-Response Pipeline

Every chat message passes through a 6-stage pipeline. The orchestrator (orchestrator.py — 1,085 LOC) coordinates all stages.

1
Input Sanitization & Chitchat Detection
Control characters stripped, message capped at 500 chars. _CHITCHAT_RE regex detects greetings, compliments, and affirmations — these skip the entire context assembly pipeline for instant response. _sanitize_assistant_text() strips any leaked Anthropic tool-call XML from prior responses.
2
History Loading & Compaction
Full message history loaded for session. If >12 messages, history_compactor.py summarizes older messages via fast LLM call (keeps last 4 verbatim). Summary stored in content_summary field. Hard-truncate as final fallback.
Threshold: 12 messages → compact
Keep recent: 4 messages verbatim
Summary: max 300 words via Haiku
3
Concurrent Context Assembly
Three retrieval tasks fire in parallel via asyncio.gather():
Entity Resolution — Haiku NER extracts business entities → pg_trgm fuzzy match against 2,109 tenant_entity_mappings → <tenant_vernacular> XML injection

Domain Knowledge — OpenAI text-embedding-3-small (1536-dim) embeds query → pgvector cosine search on domain_knowledge_chunks → top-K results + keyword ILIKE fallback

Proven Patterns — Embed query → pgvector cosine search on tenant_query_patterns → return SQL templates with success_count

Brand Identity — TenantConfig.brand_name injected into system prompt so AI identifies as the white-labeled brand

Soul.md — Tenant personality, tone preferences, and NetSuite quirks parsed from markdown

Financial Mode Detection — classify_intent() checks for financial report queries → augments task with verified TAL column names, CONSOLIDATE functions, mandatory filters
4
Agent Execution
Primary: Unified Agent (unified_agent.py — 560 LOC) — single agent with 10 tools, up to 10 reasoning steps. Anti-hallucination guardrail forces tool execution when step==0 and no tools called for data queries.

Fallback: Multi-Agent Coordinator (coordinator.py — 1,262 LOC) — heuristic regex classifier → LLM fallback for ambiguous queries → dispatch specialist agents (SuiteQL max 6 steps, RAG max 2 steps, Workspace, Analysis).
Tool Inventory (10 tools):
netsuite_suiteql — Execute SuiteQL queries via REST API
netsuite_get_metadata — Schema introspection & field lookup
rag_search — Vector + keyword doc chunk retrieval
web_search — Brave Search API for online research
workspace_list_files — Browse workspace file tree
workspace_read_file — Read file content
workspace_search — Full-text search across workspace
workspace_propose_patch — Generate unified diff patches
tenant_save_learned_rule — Persist user corrections
netsuite_get_record — Fetch individual NetSuite records
5
Confidence Scoring
CompositeScorer (confidence_service.py) aggregates 6 weighted signals into a 1.0–5.0 score:
LLM self-assessment (0.40) — <confidence> tag extracted from response
Query pattern similarity (0.15) — pgvector cosine match to proven patterns
Pattern success boost (0.10) — success_count / 50 (capped at 0.1)
Domain knowledge similarity (0.10) — mean cosine of top-3 chunks
Entity resolution confidence (0.15) — pg_trgm similarity score
Tool success rate (0.10) — successful / total tool calls

Penalties: -0.2 if tools needed but none called, -0.3 if any tool failed
Final: clamp(weighted_sum, 0, 1) × 4 + 1 → 1.0–5.0
6
SSE Streaming & Persistence
Response streamed via Server-Sent Events with 5 event types: text (token-by-token), tool_status (tool progress), confidence (score + explanation), error (failures), message (final complete message). ChatMessage persisted with confidence_score, token counts, model/provider metadata. Memory updater fires async (regex-gated for corrections). Successful SuiteQL patterns saved for self-learning.
SSE format: data: {"type":"text","content":"..."}\n\n
Cloudflare padding: 2KB space prefix forces stream start
Output sanitization: strips leaked <invoke>, <parameter>, <tool_use> tags

Authentication Flow
🎟
JWT Token Structure
core/security.py
  • sub — User UUID
  • exp — Expiry (30min access, 7d refresh)
  • type — "access" or "refresh"
  • jti — Unique token ID (UUID hex)
  • Signed: HS256 with JWT_SECRET_KEY
  • Access token: localStorage + cookie for middleware
  • Refresh token: HttpOnly cookie only
🚫
Token Revocation
core/token_denylist.py
  • JTI-based revocation via Redis
  • Key: jwt:denied:{jti}
  • TTL = token expiry - now (auto-cleanup)
  • Checked on every authenticated request
  • Logout → adds both access + refresh JTIs
  • Fallback: in-memory dict (dev only)
🔄
Token Refresh Flow
api-client.ts + auth_service.py
  • Frontend: 5-min interval proactive check
  • Refreshes 2 min before expiry
  • On 401: silent refresh → retry request
  • Concurrent refresh attempts deduplicated
  • Backend: old refresh JTI added to denylist
  • Failure: redirect to /login, clear all tokens
🏢
Multi-Tenant Data Isolation
Row-Level Security (RLS): Every authenticated request calls SET LOCAL app.current_tenant_id = '{uuid}' on the database session. All tenant-scoped tables enforce RLS policies matching tenant_id to this session variable. UUID is validated before interpolation (no bind params allowed by SET LOCAL).
🗄
RLS Implementation
core/database.py
  • set_tenant_context(session, tenant_id)
  • UUID validation: str(uuid.UUID(tenant_id))
  • SET LOCAL scoped to transaction
  • Every table has tenant_id indexed column
  • Prevents cross-tenant data access at DB level
🔐
Encryption at Rest
core/encryption.py
  • Fernet symmetric encryption (AES-256-CBC + HMAC)
  • OAuth access/refresh tokens encrypted
  • BYOK AI provider API keys encrypted
  • NetSuite credentials encrypted
  • Key stored in ENCRYPTION_KEY env var
  • Production: validated at startup (no defaults)

🚦
Rate Limiting
core/rate_limit.py
  • Redis-backed sliding window (sorted sets)
  • Login: 10 attempts / 60 seconds per IP
  • MCP tools: per-tool limits (2-120/min)
  • Auto-cleanup via Redis TTL expiry
  • Fallback: in-memory dict (dev only)

Input Sanitization
Multiple layers
  • Pydantic v2 validates all API request bodies
  • Chat input: control chars stripped, 500 char cap
  • MCP params: allowlist validation per tool
  • SQL injection: parameterized queries (except SET LOCAL)
  • Output: _sanitize_assistant_text() strips leaked XML
  • MCP: sensitive fields redacted (password, secret, token)
